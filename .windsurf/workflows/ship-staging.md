# /ship-staging — Staging Deploy Workflow

> Deployt eine App auf den Staging-Server (Dev Desktop 88.99.38.75).
> Nutzt das gleiche `docker-compose.prod.yml` wie Production (ADR-157 Option A).
> Unterschied: Server-IP, `.env.staging` statt `.env.prod`, Staging-Domain.

## Step 0: Parameter ermitteln

Ermittle die Deploy-Parameter:

1. **scope** — Repo-Name (z.B. `risk-hub`)
2. **staging_port** — Port aus `ports.yaml` (Feld `staging`)
3. **web_container** — Docker Container-Name (z.B. `risk_hub_web`)

// turbo
```bash
python -c "
import yaml
from pathlib import Path
p = Path('${GITHUB_DIR:-$HOME/github}/platform/infra/ports.yaml')
d = yaml.safe_load(p.read_text())
for name, cfg in sorted(d.get('services',{}).items()):
    if cfg and cfg.get('staging'):
        print(f'{name}: staging={cfg[\"staging\"]}, domain={cfg.get(\"domain_staging\",\"-\")}')"
```

## Step 0.5 — Port-Audit Gate (ADR-157)

// turbo
```bash
python ${GITHUB_DIR:-$HOME/github}/platform/infra/scripts/port_audit.py --offline
```

**Abbruch wenn:** Exit-Code != 0.

---

## Step 1 — Branch + Status verifizieren

```bash
git -C ${GITHUB_DIR:-$HOME/github}/{scope} branch --show-current
git -C ${GITHUB_DIR:-$HOME/github}/{scope} status --short
```

Erwartung: Branch = `main`, keine uncommitted Änderungen.

---

## Step 2 — Push nach GitHub

// turbo
```bash
git -C ${GITHUB_DIR:-$HOME/github}/{scope} push origin main
```

---

## Step 3 — Staging-Server vorbereiten

Prüfe ob das Repo auf dem Staging-Server existiert:

```bash
ssh root@88.99.38.75 "test -d /opt/{scope} && echo EXISTS || echo MISSING"
```

**Falls MISSING — Erstmalig klonen:**

```bash
ssh root@88.99.38.75 "git clone git@github.com:achimdehnert/{scope}.git /opt/{scope}"
```

**Falls EXISTS — Pull:**

```bash
ssh root@88.99.38.75 "cd /opt/{scope} && git pull origin main"
```

---

## Step 4 — .env.staging prüfen

```bash
ssh root@88.99.38.75 "test -f /opt/{scope}/.env.staging && echo OK || echo MISSING"
```

**Falls MISSING** — `.env.staging` Template erstellen:

```bash
ssh root@88.99.38.75 "cat > /opt/{scope}/.env.staging << 'EOF'
# Staging Environment — {scope}
# Generated by /ship-staging workflow
DJANGO_SETTINGS_MODULE=config.settings.production
SECRET_KEY=staging-secret-change-me-$(openssl rand -hex 16)
DEBUG=false
ALLOWED_HOSTS=localhost,127.0.0.1,{domain_staging}
CSRF_TRUSTED_ORIGINS=https://{domain_staging}
DATABASE_URL=postgres://{scope_underscore}:staging_pw@localhost:5432/{scope_underscore}_staging
REDIS_URL=redis://localhost:6379/1
EOF"
```

⚠️ **User muss DB-Passwort und ggf. weitere Secrets manuell setzen!**

---

## Step 5 — Docker Compose Up (Staging)

```bash
ssh root@88.99.38.75 "cd /opt/{scope} && docker compose -f docker-compose.prod.yml --env-file .env.staging pull && docker compose -f docker-compose.prod.yml --env-file .env.staging up -d"
```

---

## Step 6 — Migrate

```bash
ssh root@88.99.38.75 "cd /opt/{scope} && docker compose -f docker-compose.prod.yml exec -T web python manage.py migrate --noinput"
```

---

## Step 7 — Health Check

```
mcp0_ssh_manage:
  action: http_check
  host: 88.99.38.75
  url: http://127.0.0.1:{staging_port}/livez/
  expect_status: 200
```

Bei HTTP 200 → Staging-Deploy erfolgreich.

**Discord-Notification:**

```
mcp2_discord_notify:
  title: "🧪 Staging Deploy: {scope}"
  message: "Staging-Deploy auf 88.99.38.75:{staging_port} erfolgreich. Domain: {domain_staging}"
  level: success
```

## Fehlerbehebung

| Problem | Lösung |
|---------|--------|
| Container crasht | `ssh root@88.99.38.75 "docker logs {web_container} --tail 50"` |
| DB fehlt | `ssh root@88.99.38.75 "sudo -u postgres createdb {scope_underscore}_staging"` |
| Port belegt | `ssh root@88.99.38.75 "ss -tlnp | grep {staging_port}"` |
| .env fehlt | Step 4 wiederholen |
